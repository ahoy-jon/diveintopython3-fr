Lançons Nous

Avant de pouvoir commencer à programmer avec Python 3, vous devez l’installer. Ou pas ?

Quel Python est votre Python ?

Si vous utilisez un compte sur un serveur d’hébergement, votre hébergeur a peut-être déjà installé Python 3. Si vous utilisez Linux à la maison, you avez peut-être aussi déjà Python 3. La plupart des distributions GNU/Linux embarquent Python 2 par défaut; un petit nombre de distributions incluent Python 3 et cette tendance s’accélère. Mac OS X inclue une version en ligne de commande de Python2, but au moment où ce livre s’écrit il n’inclue pas Python 3. Quant à Microsoft Windows il n’inclue aucune version de Python. Mais pas d’affolement ! Il est toujours possible d’installer Python en quelques cliques, peu importe le système d’exploitation que vous utilisez.

La manière la plus simple pour vérifier si Python 3 est sur votre Linux ou votre Mac OS X est d’accéder Ã  la ligne de commande. Une fois que vous avez atteint la ligne de commande de votre système, tappez juste python3 (tout en minuscule, sans espaces), appuyer sur la touche ENTRER, et regarder ce qui se passe. Sur mon ordinateur sous Linux, Python 3.1 est déjà installé, et cette commande m’envoit dans le shell interactif de Python. 

mark@atlantis:~$ python3
Python 3.1 (r31:73572, Jul 28 2009, 06:52:23) 
[GCC 4.2.4 (Ubuntu 4.2.4-1ubuntu4)] on linux2
Type "help", "copyright", "credits" or "license" for more information.
>>>

(Tappez exit() et appuyez sur ENTRER pour sortir du shell interactif de Python.)

Mon hébergeur tourne aussi sous Linux et me donne l'accès à une ligne de commande, mais mon serveur ne possède pas Python 3 d’installé. (Sniff!) 

mark@manganese:~$ python3
bash: python3: command not found

Donc pour en revenir à la question initiale, “Quel Python est votre Python ?” Celui qui tourne sur l’ordinateur que vous avez déjà.

Installation sous Microsoft Windows

Windows vient sous deux formes d’architectures ces temps-ci: 32-bit et 64-bit. Bien sûr, il y a beaucoup de différences entre les versions Windows — XP, Vista, Windows 7 — mais Python tourne indifférement sur toutes ces versions. La plus importante distinction vient de l’architecture entre 32-bit et 64-bit. Si vous n’avez aucune idée de l’architecture que vous utilisez, alors c’est probablement du 32-bit.

Allez sur python.org/download/ et téléchargez la version appropriée de Python 3 pour votre architecture. Vos choix possibles ressembleront à quelque chose comme ça :

  • Python 3.1 Windows installer (Windows binary — does not include source)
  • Python 3.1 Windows AMD64 installer (Windows AMD64 binary — does not include source)

Je refuse d’inclure un lien direct sur cette page, car des mise à jour mineure arrivent tout le temps et je ne veux pas ê accuser de vous avoir faire manquer une mise à jour importante. Vous devez toujours installer la plus récente version de Python 3.X sauf si vous possèdez une bonne raison esothérique de ne pas le faire.

  1. Une fois votre téléchargement terminé, double-cliquez sur le fichier .msi. Windows va ouvrir une alerte de sécurité, vu que vous êtes sur le point d’exécuter un programme. L’installeur officiel de Python possède une signature digitale signée par la Python Software Foundation, l’organisation à but non-commerciale qui gère le développement de Python. Refusez toute imitations !

    Cliquez sur le bouton Exécuter pour lançer l’installation de Python 3.

  2. La première question que l’;installeur va vous poser est si vous souhaitez installer Python 3 pour tous les utilisateurs ou juste pour vous. Le choix par défaut propose de “l’installer pour tous les utilisateurs,” ce qui est le meilleur choix sauf si vous avez une bonne raison pour ne pas vouloir. (Une bonne raison pour “installer juste pour vous” est si vous installer Python sur l’ordinateur de votre société et que vous n’avez pas les droit d’administrateur sur votre compte Windows. Mais si c’est vraiment le cas, pourquoi donc installez-vous Python sans la permission de votre administrateur Windows ? A la réflexion, je ne veux pas savoir ! Je ne veux pas d’ennuis !)

    Cliquez sur le bouton Suivant pour valider votre type d’installation.

  3. Ensuite, l’installeur va vous demander de choisir une destination où installer Python 3. L’emplacement par défaut de toutes les versions de Python 3.1.x est C:\Python31\, ce qui devrait fonctionner pour la plupart des utilisateurs lambda sauf si une raison spécifique vous demande de changer cet emplacement. Si vous avez définit une partition annexe pour installer certaines applications (e.g. D:), vous pouvez sélectionner ce chemin avec les boutons de l’interface graphique, ou simplement entrer le chemin en toute lettre au clavier. Vous n’êtes donc pas forcé d’installer Python sur le lecteur C:; vous pouvez l'installer sur n'importe quel lecteur et dans n'importe quel dossier.

    Cliquez sur le bouton Suivant pour valider votre destination d’installation.

  4. La page suivante semble compliquée mais ne l’est pas vraiment. Comme avec beaucoup d’installeurs, vous avez la possibilité de ne pas installer tous les composants de Python 3. Ainsi si votre espace disque disponible est très faible, vous pouvez exclure de l’installation certains composants.

    • L’option Register Extensions vous permet de double-cliquer sur des scripts Python (fichiers .py) pour les exécuter. Elles est recommandé mais non nécessaire. (Cette option ne consomme pas d’espace disque, il y a donc peu d’intérêt à l’exclure de l’installation.)
    • Tcl/Tk est la bibliothèque graphique utilisée par le shell Python, que nous allons utiliser à travers cet ouvrage. Je recommande très fortement de conserver cette option.
    • L’option Documentation installe des fichiers d’aide qui contiennent presque autant d’informations que docs.python.org. Cette option est recommandé surtout si votre accès à Internet est limité.
    • L’option Utility Scripts inclue le très utile script 2to3.py dont nous allons apprendre l’utilisation plus tard dans cet ouvrage. Cette option est nécessaire si vous voulez apprendre à migrer des projet Python 2 vers Python 3. Si vous n’avez aucun code Python 2 existant, vous pouvez vous passer de cette option.
    • L’option Test Suite installe une collection de scripts pour tester l’interpréteur Python lui-même. Nous n’allons pas l'utiliser dans cet ouvrage, je ne l’ai d’ailleurs jamais utilisé dans le cadre d’aucun cours de programmation avec Python. Considérons le donc comme complètement optionnel.

  5. Si vous n’êtes pas sûr de combien d’espace disque vous disposez, cliquez sur le bouton Espace Libre. L’installeur fera la liste de vos lecteurs, calculera l’espace disponible sur chacun d’eux et l’espace qu’il resterait sur chacun d’eux après l’installation.

    Cliquez sur le bouton OK pour retourner sur la page de “Personnalisation de votre installation Python”.

  6. Si vous décidez de supprimer une option, il suffit de selectionner le menu déroulant associé à l’option et de sélectionner “La fonctionnalité entière sera indisponible.” Par exemple, exclure les outils de tests vous fera sauver quelques 7908KB d’espace disque.

    Cliquez sur le bouton Suivant pour valider définitivement vos choix d’options.

  7. L’installeur va copier tous les fichiers nécessaires dans votre dossier de destination. (C’est arrivé si vite chez moi que j’ai dû m’y reprendre à trois fois juste pour avoir une copie d’écran correcte !)

  8. Cliquez sur le bouton Terminé pour fermer l’installeur.

  9. Dans votre menu Démarrer devrait se trouver maintenant un nouveau menu appelé Python 3.1. Avec à l’intérieur, un programme appelé IDLE. Sélectionnez celui-ci pour lancer le shell interactif de Python.

Installation sous Mac OS X

Tous les ordinateurs Macintosh modernes utilisent un processeur Intel (comme la plupart des PC Windows). Les anciens Macs utilisent des processeurs PowerPC. Vous n’avez même pas besoin de connaître la différence, car il n’y a qu’un seul installeur Mac pour Python et ce pour tous les Macs.

Aller sur le site python.org/download/ et télécharger l’installeur Python pour Mac. Il sera appelé quelque chose comme Python 3.1 Mac Installer Disk Image, même si le numéro de version peut varier. Soyez juste sûr de télécharger une version 3.x, et pas 2.x.

  1. Votre navigateur internet devrait monter automatiquement l’image disque et ouvrir une fenêtre Finder pour vous en montrer le contenu. (Si cela n’arrive pas, il vous faudra aller chercher dans votre dossier de téléchargement et double-cliquer sur l’image dmg pour la monter. Elle sera nommé quelque chose comme python-3.1.dmg.) L’image disque contient un certain nombre de fichiers texte (Build.txt, License.txt, ReadMe.txt), et notre objectif l’installeur, Python.mpkg.

    Double-cliquez sur l’installeur Python.mpkg pour lancer l’installation de Python pour Mac.

  2. La première page de cet installeur vous donne une bref description de Python en lui-même, puis vous renvoie au fichier ReadMe.txt (que vous n’avez pas lu, hein ?) pour plus de détails.

    Cliquez sur le bouton Continuer pour la suite.

  3. La page suivante contient en revanche quelques informations importantes : Python nécessite Mac OS X 10.3 ou supérieure. Si vous utilisez toujours Mac OS X 10.2, vous devriez vraiment mettre votre système à jour. Apple ne fournit d’ailleurs plus de mises à jour de sécurité pour votre système d’exploitation, ainsi il est risqué d’utiliser votre ordinateur pour aller sur Internet. En plus vous ne pourrez pas exécuter Python 3.

    Cliquez sur le bouton Continuer pour avancer.

  4. Comme tout bon installeur, celui de Python vous demande d’accepter la licence d’utilisation. Seulement Python est libre et open source, et la licence qu’il utilise est validée par l’OSI ( Open Source Initiative ). Python a eu dans son histoire un certain nombre de propriétaires et de sponsors, chacun ayant laissé son empreinte sur la licence du logiciel. Mais la conclusion de cette histoire est simple : Python est libre et open source, et vous pouvez l’utiliser sur n’importe quelle plateformes, pour n’importe quel motif, et sans frais ni obligations.

    Cliquez sur le bouton Continuer encore une fois.

  5. À cause du processus d’installation standard d’Apple, vous devez “accepter” la licence du logiciel pour pouvoir terminer l’installation. Mais étant donné que Python est open source, vous n’êtes jamais qu’en train de confirmer que la licence vous donne tout les droits précédemment évoqués, plutôt que de vous les enlever.

    Cliquez sur le bouton Accepter pour continuer.

Installation sous Ubuntu Linux

Les distributions modernes de Linux possèdent tous de larges dépôt d'applications précompilées, prêtes à être installées. Les détails d'implémentation varient cependant selon les distributions. Sous Ubuntu Linux, la façon la plus simple d'installer Python 3 est d'utiliser l'outil Ajouter/Supprimer de votre menu Applications.

  1. Quand vous lancez pour la première fois l'outil Ajouter/Supprimer, il vous montre une liste d'applications pré-sélectionnée dans différentes catégories. Certaines sont déjà installées; la plupart ne le sont pas. Comme les dépôts contiennent plus de 10 000 applications, il y a différents filtres que vous pouvez appliquer pour voir seulement des parties des dépôts. Le filtre par défaut inclue les “applications maintenues par Canonical,” ce qui correspond à un ensemble relativement restreint des applications officiellement supportée par Canonical, la société qui a créé et maintient Linux.

  2. Python 3 n'est pas maintenu par Canonical, donc la première étape est de changer le filtre par défaut pour sélectionner le filtre “Toutes applications Open Source.”

  3. Une fois votre filtre élargit pour inclure toutes applications open source, utilisez le champ de Recherche juste après le menu Filtre pour chercher Python 3.

  4. Ainsi la liste des applications se réduit à ceux dont les critères correspondent au mots clés Python 3. Nous allons donc récupérer deux paquets. Le premier est Python (v3.0). Il contient l'interpréteur Python en lui-même.

  5. Le second paquet que nous allons choisir est directement au dessus : IDLE (avec Python-3.0). C'est le shell interactif Python que nous allons tout le temps utiliser à travers cet ouvrage.

    Après avoir sélectionné ces deux paquets, cliquez sous le bouton Appliquez les Changements pour continuer.

  6. Le gestionnaire de paquets va vous demander de confirmer votre choix d'installer à la fois IDLE (avec Python-3.0) et Python (v3.0).

    Cliquez sur le bouton Appliquer pour continuer.

  7. Le gestionnaire de paquets va afficher une barre de progression pendant qu'il télécharge les données nécessaire depuis les dépôts Internet de Canonical.

  8. Une fois les données téléchargées, le gestionnaire de paquets va automatiquement commencer l'installation.

  9. Si tout se passe bien, le gestionnaire de paquets va confirmer que l'installation a été un succès. A partir de là, vous pouvez double-cliquer sur l'application IDLE pour lancer le shell Python, ou cliquer sur le bouton Terminé du gestionnaire de paquets.

    Vous pouvez toujours relancer le shell Python en allant dans le menu Applications, puis dans la sous-menu Programmation, et sélectionner IDLE.

  10. Le shell Python est l'endroit où vous passerez le plus clair de votre temps d'exploration de Python. Les exemples de cet ouvrage partiront d'ailleurs du principe que vous pouvez vous y retrouver dans le shell Python, donc autant s'entraîner !

Installation sous d’Autres Plateformes

Python 3 est disponible sur un certain nombre de plateformes. En particulier, il est disponible sur virtuellement toutes les distributions basées sur Linux, BSD, ou Solaris. Par exemple, RedHat Linux utilise le gestionnaire de paquet yum. FreeBSD a une collection de paquets disponibles, SUSE a zypper, et Solaris pkgadd. Une recherche rapide sur Internet avec comme mots clés Python 3 + votre système d'exploitation devrait vous dire si un quelconque paquet de Python 3 est disponible, et si oui, comment l'installer.

Utilisation du shell Python

Le shell Python est l'endroit où l'on peut explorer la syntaxe de Python, avoir de l'aide de manière interactive sur des commandes, et débogguer des petits programmes. Le shelle graphique de Python (appellé IDLE) contient en plus un éditeur de texte déçent qui supporte une coloration syntaxique du code Python et s'intègre bien avec le shell Python. Ainsi si vous n'avez pas déjà un éditeur de texte fétiche, vous pouvez toujours essayer IDLE.

Commençons par le commencement. Le shell Python en lui-même est un superbe terrain de jeu interactif. À travers cet ouvrage, vous allez voir des exemples de ce type : 

>>> 1 + 1
2

Les trois symboles supérieurs, >>>, symbolisent le shell Python. Ne tapez jamais cette partie. C'est juste pour vous faire comprendre que cet exemple doit être exécuté depuis le shell Python.

1 + 1 est donc la partie que l'on tape dans le shell. Vous pouvez taper n'importe quelle valide expression Python ou commande dans le shell Python. Ne soyez pas timide, il ne mords pas ! Le pire qui puisse arriver est d'obtenir un message d'erreur. Les commandes sont par ailleurs exécutées immédiatement (une fois la touche ENTRÉE pressée); et les expressions sont évaluées immédiatement, et le shell Python affiche le résultat à l'écran.

2 est le résultat de l'évaluation de cette expression. Comme il est clair que, 1 + 1 est une expression Python valide. Le résultat, bien sûr, est 2.

Essayons autre chose. 

>>> print('Hello world!')
Hello world!

C'est assez simple, non ? Mais il y a plus à faire dans le shell Python que cela. Si à un moment vous êtes bloqué — vous n'arrivez pas à vous souvenir d'une commande, ou vous n'arrivez plus à remettre la main sur les arguments à passer à une certaine fonction — vous pouvez toujours obtenir de l'aide de manière interactive avec le shell de Python. Tapez juste help et appuyez sur ENTRÉE. 

>>> help
Type help() for interactive help, or help(object) for help about object.

Il y a deux modes d'aide. Vous pouvez obtenir de l'aide à propos d'un seul objet, ce qui va juste vous afficher la documentation et vous rendre immédiatement la main. Vous pouvez aussi rentrer dans le mode d'aide, où au lieu d'évaluer des expressions Python, vous tapez des mot clés ou des noms de commandes et le shell affichera à l'écran tout ce qu'il sait sur cette commande.

Pour entrer dans ce mode interactif, tapez help() et appuyez sur ENTRÉE. 

>>> help()
Welcome to Python 3.0!  This is the online help utility.

If this is your first time using Python, you should definitely check out
the tutorial on the Internet at http://docs.python.org/tutorial/.

Enter the name of any module, keyword, or topic to get help on writing
Python programs and using Python modules.  To quit this help utility and
return to the interpreter, just type "quit".

of what it does; to list the modules whose summaries contain a given word
such as "spam", type "modules spam".

help>

Cela permet de bien vous rappeller que vous êtes dans le mode d'aide interactive de Python et pas dans l'invite de commande normal. Maintenant vous pouvez entrer n'importe quel mot clé, commande, nom de module, nom de fonction — en fait tout ce que Python peut arriver à comprendre — et obtenir directement la documentation associée. 

help> print                                                                 ①

  1. Pour obtenir la documentation sur la fonction print(), il vous suffit de taper print et d'appuyer sur ENTRÉE. Le mode d'aide interactif va alors afficher une page d'aide ressemblant très fortement à une page man sous Linux avec : le nom de la fonction, un léger résumé de ses fonctionnalités, les arguments d'entrée de la fonction ainsi que leurs valeurs par défauts, et ainsi de suite. Si la documentation semble légèrement abstraite pour vous, ne paniquez pas. Vous apprenderez plus sur les concepts de base de Python dans les prochains chapitres.
  2. Bien sûr , le mode d'aide interactif ne sait pas tout. Si vous tapez quelque chose qui n'est pas une commande Python, un module, une function, or tout autre mot clé du cœur de Python, le mode d'aide interactive va juste s'avouer vaincu et dire qu'il ne peut rien pour vous.
  3. Pour quitter le mode d'aide interactif , il suffit de taper quit et d'appuyer sur la touche ENTRÉE.
  4. L'invite de commande reviendra à son état normal >>> pour vous confirmer que vous avez quitter le mode d'aide interactif et que vous êtes maintenant de retour dans le shell Python.

IDLE, le shell graphique de Python, inclue aussi un éditeur de texte adapté à Python.

Les éditeurs Python et IDEs

IDLE n'est pas le seul logiciel utilisable quand il s'agit d'écrire des programmes en Python. Car si c'est un bon éditeur quand il s'agit d'apprendre sur le langage Python en lui-même, de nombreux développeurs préfèrent utiliser d'autres éditeurs de texte ou d'autres Environnements de Développement Intégrés (des IDE (Integrated Development Environments)). Je ne vais pas couvrir ce sujet ici, mais la communauté Python maintient une liste des éditeurs de textes utilisables avec Python qui permet de trouver son bonheur à la fois en terme de plateforme et en terme de licenses logicelles.

Vous voudrez aussi probablement regarder la liste des IDEs adaptés à Python, même si très peu d'entre eux à l'heure actuelle supportent Python 3. Par exemple PyDev, qui est un plugin pour Eclipse et qui transforme Eclipse en un très bon IDE pour Python, supporte maintenant Python 3. En plus à la fois Eclipse et PyDev sont multi-platformes et open source.

Du coté commerciale, il y a l'IDE Komodo d'ActiveState. Sa license est disponible par utilisateur, mais les étudiants peuvent obtenir un tarif réduit, et une version gratuite limitée dans le temps est disponible.

Je programme en Python depuis maintenant neuf ans, et je créé mes programmes Python principalement avec GNU Emacs en déboggant avec le shell Python. Il y a pas de bonne ou de mauvaise façon de développer en Python. Trouvez celui qui vous convient !
